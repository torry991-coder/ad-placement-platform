## 2. 快速开始

### 2.1 环境要求

- Python 3.11+
- Node.js 18+ (前端)
- Docker (可选，生产部署)

### 2.2 开发模式启动

```bash
# 1. 后端
cd 智能广告投放系统
pip install -r backend/requirements.txt
```

## 5. 核心引擎详解

### 5.1 竞价引擎 (`bidding_engine.py`)

**工作流程:**
1. 接收拍卖上下文 (活动、受众、设备、平台)
2. ML 引擎预测 CTR / CVR
3. 根据出价策略计算最优出价:
   - `max_conversions`: 基于预测 CVR 调整，预算内最大化转化
   - `target_cpa`: 出价 = 目标 CPA × 预测 CVR
   - `target_roas`: 出价 = 预估转化价值 / 目标 ROAS
   - `enhanced_cpc`: 基准 CPC × ML 调整系数
   - `manual_cpc`: 固定 max_cpc
4. 预算检查 (pacing 状态)
5. 模拟拍卖竞争 (竞争出价 + 随机噪音)
6. 返回结果 (bid_amount, won, win_price, ad_rank)

**模型回退**: XGBoost 不可用 → 统计模型 (行业基准 CTR/CVR)

### 5.2 ML 预测引擎 (`ml_engine.py`)

**输入特征:**
- 活动级: bid_strategy, daily_budget, target_cpa, target_roas
- 受众级: age_range, gender, device, platform
- 时间级: hour, day_of_week
- 上下文: budget_spent_today, budget_remaining_ratio

**模型选择:**
1. XGBoost (需安装 xgboost 包) — 训练后模型
2. 线性回归 (scikit-learn) — 备用模型
3. 统计回退 — 行业基准值 (CTR≈3%, CVR≈4%)

**缓存**: 相同特征的预测结果缓存 60 秒

### 5.3 预算管理 (`budget_pacer.py`)

**Pacing 状态:**
- `on_track`: 花费在预期范围的 ±15% 内
- `overspending`: 超出预期 15%+ → 降低出价
- `underspending`: 低于预期 15%+ → 提高出价

**时段加权**: 按小时分配权重 (0-6点低峰, 10-12点/19-22点高峰)

### 5.4 归因模型 (`attribution.py`)

| 模型 | 说明 |
|------|------|
| `last_touch` | 最后一次触点 100% 归因 |
| `first_touch` | 第一次触点 100% 归因 |
| `linear` | 所有触点平均分配 |
| `time_decay` | 越近触点权重越高 |
| `position_based` | 首尾各40%，中间20% |
| `data_driven` | Shapley 值计算各触点边际贡献 |

### 5.5 创意轮换 (`creative_rotator.py`)

**疲劳度计算:** 基于展示次数、CTR 趋势、最近展示时间
- 疲劳度 > 70: 自动暂停
- 疲劳度 40-60: 降低展示频率
- 疲劳度 < 30: 正常投放

### 5.6 告警引擎 (`rule_engine.py`)

**支持的指标:** impressions, clicks, conversions, spend, revenue, ctr, cvr, cpc, cpa, roas

**支持的操作符:** `>`, `<`, `>=`, `<=`, `==`, `!=`

**动作类型:**
- `notify`: UI 通知 + Webhook
- `pause_campaign`: 自动暂停活动
- `reduce_budget`: 自动削减预算 20%

**冷却时间:** 可配，默认 60 分钟

## 6. AI 智能体系统

### 6.1 6-Agent 管道

```
Auditor → Strategist → BudgetAgent → AudienceAgent → CreativeAgent → ReportAgent
 审计       策略         预算             受众             创意            报告
```

| Agent | 功能 | 输出 |
|-------|------|------|
| Auditor | 活动健康审计 | 评分(0-100)、问题列表、建议 |
| Strategist | 出价/预算策略 | 策略调整建议、预算重分配 |
| BudgetAgent | 预算 pacing 分析 | 时段调整、跨平台再分配 |
| AudienceAgent | 受众扩展 | Lookalike 分组、扩展策略 |
| CreativeAgent | 创意生成 | 3个头条+描述+CTA 变体 |
| ReportAgent | 执行摘要 | Markdown 格式报告 |

### 6.2 LLM 支持

| Provider | 需要 | 模型 |
|----------|------|------|
| OpenAI | `OPENAI_API_KEY` | gpt-4o / gpt-4o-mini |
| Google | `GOOGLE_API_KEY` | gemini-2.0-flash |
| Ollama | `OLLAMA_BASE_URL` | llama3 (本地) |
| Fallback | 无需配置 | 规则引擎 (始终可用) |

### 6.3 使用方式

```bash
# API 调用
curl -X POST http://localhost:8000/api/agent/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"分析活动1的ROAS表现","campaign_id":1}'

# SSE 流式
curl -N http://localhost:8000/api/agent/stream?message=优化我的活动预算
```

## 9. 常见问题

### Q: 前端启动报 "npm: command not found"
- 确保 Node.js 已安装并在 PATH 中
- Windows: 检查 `F:\Node.js` 是否在系统环境变量中

### Q: 后端启动报数据库连接错误
- SQLite: 确保 `ad_platform.db` 文件可写
- PostgreSQL: 检查 `DATABASE_URL` 和 Docker 容器状态

### Q: ML 预测返回 "statistical" 模型
- XGBoost 未安装: `pip install xgboost`
- 或者系统使用统计回退，结果仍然有效

### Q: AI Agent 返回规则引擎结果
- 未配置 LLM API Key → 自动使用规则回退
- 配置 `OPENAI_API_KEY` 或 `GOOGLE_API_KEY` 启用 LLM

### Q: 如何切换到 PostgreSQL
```bash
# .env
DATABASE_URL=postgresql+asyncpg://adplatform:adplatform@localhost:5432/ad_platform
```
Docker Compose 已包含 PostgreSQL 服务。

### Q: 如何重置所有数据
```bash
rm ad_platform.db          # 删除 SQLite 数据库
python -m backend.seed     # 重新生成种子数据
```
