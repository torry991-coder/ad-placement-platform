# 智能广告投放系统 — 竞品调研与修改建议

> 调研时间: 2026-06-14  
> 调研范围: GitHub 开源项目、商业化产品、技术博客  
> 对标项目: openadserver (196⭐), AIAdPlacer, Lorn.ADSP, 创量智投

---

## 一、竞品概览

### 1. openadserver (⭐196，Apache 2.0)
- **技术栈**: Python/FastAPI + PyTorch + PostgreSQL + Redis
- **核心亮点**: DeepFM 深度学习 CTR 预测，<10ms P99 延迟，<5ms 推理延迟
- **与我们的差异**: 使用 DeepFM 替代 XGBoost；有 Prometheus + Grafana 监控；有事件追踪端点

### 2. AIAdPlacer (AI Native pDOOH)
- **技术栈**: Python/FastAPI + LangGraph + Ollama + ChromaDB + PostgreSQL
- **核心亮点**: RAG 知识库、腾讯地图 LBS 归因、MCP 接口（AI-to-AI）、竞品监控、公交线路竞价子系统
- **与我们的差异**: 有 RAG 知识库；有 LBS 地理归因；有竞品监控；符合 T/CCSA 行业标准

### 3. Lorn.ADSP (C#/.NET，60+ commits)
- **技术栈**: C#/F# + SQL Server + Redis + RabbitMQ + Docker/K8s
- **核心亮点**: 完整 CRM + 合同管理、广告位库存管理、DSP/SSP/ADX 接入、多租户 SaaS
- **与我们的差异**: 有 CRM 和合同管理；有广告位库存系统；有 DSP/SSP 标准化对接

### 4. 创量智投 (商业化产品)
- **功能**: 批量创建 1000+ 计划、跨媒体多账户聚合、24H AI 托管、素材资产管理
- **覆盖平台**: 巨量引擎、巨量千川、腾讯广告、小红书聚光、B站营销等
- **与我们的差异**: 有真实平台 API 对接；有批量操作能力；有 AI 风险控管

#### FP1. Dashboard 增加图表组件（对标商业模板 Eclan + AI Dashboard）

**现状**: Dashboard 只有 8 个 StatCard，无任何图表  
**对标**: 商业广告模板 Eclan 有 Chart.js/ApexCharts 多维图表；AI Dashboard 有 Line+Bar+Pie  
**建议**: 新建 `frontend/src/components/dashboard/` 目录：

```
components/dashboard/
├── TrendChart.tsx        # 7日 CTR/CVR/ROAS 趋势折线图 (Recharts)
├── PlatformPie.tsx       # 各平台花费占比饼图
├── BudgetGauge.tsx       # 预算利用率仪表盘
├── HourlyHeatmap.tsx     # 24h×7d 时段热力图
├── TopCampaignsBar.tsx   # Top5 活动 ROAS 柱状图
├── AlertTimeline.tsx     # 告警时间线
└── AuctionWaterfall.tsx  # 竞价瀑布图
```

实现代码示例 (TrendChart)：
```tsx
// frontend/src/components/dashboard/TrendChart.tsx
import { LineChart, Line, XAxis, YAxis, CartesianGrid, Tooltip, ResponsiveContainer, Legend } from 'recharts';

export function TrendChart({ data }: { data: TrendPoint[] }) {
  return (
    <div className="card p-5">
      <h3 className="text-sm font-semibold mb-4">7日关键指标趋势</h3>
      <ResponsiveContainer width="100%" height={300}>
        <LineChart data={data}>
          <CartesianGrid strokeDasharray="3 3" stroke="#374151" />
          <XAxis dataKey="date" stroke="#9ca3af" fontSize={12} />
          <YAxis stroke="#9ca3af" fontSize={12} />
          <Tooltip contentStyle={{ background: '#1f2937', border: '1px solid #374151' }} />
          <Legend />
          <Line type="monotone" dataKey="ctr" stroke="#3b82f6" name="CTR %" dot={false} />
          <Line type="monotone" dataKey="cvr" stroke="#10b981" name="CVR %" dot={false} />
          <Line type="monotone" dataKey="roas" stroke="#f59e0b" name="ROAS" dot={false} />
        </LineChart>
      </ResponsiveContainer>
    </div>
  );
}
```

#### 1. 添加 DeepFM / 深度 CTR 预测模型
**对标**: openadserver 用 DeepFM + LR + FM 三模型，<5ms 推理  
**现状**: XGBoost + 统计回退  
**建议**: 在 `ml_engine.py` 中增加一个 `DeepFMModel` 类，用 PyTorch 实现（或简化版用 sklearn MLP + FM），保持 XGBoost 作为备选。这是项目最大的技术亮点提升。

```python
# 新文件: backend/services/deepfm_model.py
class DeepFMModel:
    """DeepFM: FM + DNN for CTR/CVR prediction."""
    def __init__(self, feature_dims, embedding_dim=16, hidden=[256,128,64]):
        ...
    def predict(self, features) -> float:
        ...
```

#### 2. 添加 Grafana/Prometheus 监控
**对标**: openadserver 有 `/metrics` 端点 + Grafana dashboard  
**建议**: 在 `main.py` 中添加 `prometheus_fastapi_instrumentator`，暴露 QPS、延迟分布、错误率、DB 连接池利用率、缓存命中率。

```bash
pip install prometheus-fastapi-instrumentator
```

```python
# backend/main.py
from prometheus_fastapi_instrumentator import Instrumentator
instrumentator = Instrumentator()
instrumentator.instrument(app).expose(app, endpoint="/api/metrics")
```

#### 3. 事件追踪端点 (Impression/Click Tracking)
**对标**: openadserver 在广告响应中附带 `impression_url` 和 `click_url`  
**建议**: 添加 `/api/event/track?type=impression&req=xxx&ad=xxx` 端点，记录真实的展示和点击事件，区别于模拟数据。
